# Seed — Static-First FastAPI Framework

A minimal, opinionated starting point for web projects. Pages are HTML
fragments compiled into a shared shell. APIs are auto-discovered FastAPI
routers. Claude maintains its own documentation via `CLAUDE.md` files
indexed below.

**What this is:** the bones of a site or internal tool. Good for marketing
sites, dashboards, small SaaS, and client work where you want a proven
shape without a heavyweight framework.

**What this is not:** an app framework with auth/users/billing baked in.
Add those when you need them.

## Core Conventions

### Pages

Each page is a directory:

```
pages/
  home/
    config.json      # { "title": "...", "description": "...", "slug"?: "..." }
    content.html     # HTML fragment for the page body
    api.py           # Optional — FastAPI router, auto-mounted at /api/home
```

`python3 compile.py` walks `pages/`, splices each `content.html` into
`core/templates/shell.html`, and writes `output/{slug}.html`. The server
serves `output/` as static.

### APIs

Any file named `api.py` (in `pages/{slug}/`) or `*.py` (in `core/api/` or
`{any_top_level}/api/`) that exports `router = APIRouter()` is
auto-mounted at startup. No manual registration. See
`.claude/rules/api-conventions.md`.

### Docs

**Every directory that has non-obvious behaviour gets a `CLAUDE.md`.**
They exist for Claude (and future-you), not for end users. The index
below is auto-generated by `scripts/compile_docs.py` — do not hand-edit
the block between the sentinels.

Do not create standalone `.md` files. There is no `docs/` directory.

## Optional: Core / Product Split

By default everything lives in `pages/` and `core/`. When a project grows
to host **multiple independent products on one infra**, introduce
top-level product directories alongside `pages/`:

```
core/            — shared infra (templates, static, core/api/*)
pages/           — default product's pages (or shared pages like /login)
product-a/       — product A's pages/ and api/
product-b/       — product B's pages/ and api/
```

Rules of thumb if you do this:
- Products consume from `core/`, never from each other.
- Anything shared by 2+ products moves to `core/`.
- The "extraction test": you should be able to delete a product directory
  and have the rest still run.

Most projects never need this. Start without it. The server already
supports the pattern — see `discover_apis()` in `server.py`.

## Key Commands

```bash
python3 compile.py                    # Rebuild pages
python3 scripts/compile_docs.py       # Rebuild the docs index below
python3 server.py                     # Run locally on :8080
docker compose up -d                  # Docker — behind a reverse proxy (binds 127.0.0.1:${HOST_PORT})
docker compose --profile standalone up -d   # Docker — standalone with Caddy/TLS on 80/443
pytest                                # Run tests
```

## Extending the Seed

- New page: `.claude/commands/new-page.md`
- New API: `.claude/commands/new-api.md`
- Recompile after any CLAUDE.md or pages/ change: `.claude/commands/compile.md`

Path-scoped rules in `.claude/rules/` auto-activate when you touch
matching files.

## Before Every Commit

1. Update the relevant directory's `CLAUDE.md` if you changed its shape
2. Run `python3 compile.py` if pages changed
3. Run `python3 scripts/compile_docs.py` if any `CLAUDE.md` changed
4. Run `pytest` — all smoke tests must pass
